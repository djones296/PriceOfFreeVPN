There are two main types of VPN Corporate and Consumer, this tutorial will specifically focus on Consumer VPNs as they incorporate the same basic philosophy as Corporate VPNs but are open to the general public whereas corporate VPNS are only open to specified people.

With a consumer VPN the user is connected to a private network via an encrypted tunnel known as a VPN tunnel. The data which would normally be transferred through their regular internet connection will instead be encrypted and transferred through this tunnel. This step means that the data will be prevented from being intercepted and being mined by a third party. The data being transferred via the VPN tunnel will be sent to the VPN server which then forwards the required data to the internet. This step means that the users ISP can no longer see what data is being transferred from their device they will only see that the user has connected to a private server.

Although a VPN service is incredibly useful in helping to improve data privacy whilst online if connection speed is of paramount importance to you it may not be something you wish to consider using. VPN connections are generally speaking slower than regular connections largely due to three factors:

1.       The process of encrypting the data to be transferred although minimal for each piece of data does add up over time to make a noticeable difference in connection speeds.

2.       The transmission to the VPN server normally takes much longer due to its location geographically. When connecting to the server of the ISP it is safe to assume that they have a server relatively close to the user as if they had no server within a reasonable distance they would simply not offer to provide service in that area. In the case of a VPN however there is no guarantee that the server is held anywhere remotely close to the user and in many cases will be held in a different country. This extra travel distance for the data being transferred will potentially greatly slow down connection speeds depending on how far the VPN server is from each individual user.

3.       Verse With the data being encrypted it therefore must at some stage be decrypted. This adds to the slowing of the network connection for the reverse reason of the encryption of the data. Once the data reaches its final location it must be decrypted to be used for its intended purpose. Like with encryption the time taken for this process for each transfer of data is negligible but with more data being transferred this will start to become much more noticeable.
